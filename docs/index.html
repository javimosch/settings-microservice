Example Request
curl -X GET \
  http://localhost:3000/api/settings/global \
  -H 'Authorization: Bearer <token>' \
  -H 'X-Organization-Id: <org-id>' \
  -H 'X-Auth-Name: default'

# Response
[
  {
    "settingKey": "max_users",
    "settingValue": 100
  }
]

Use cases

Where this microservice fits

Designed for SaaS and platform teams that need central configuration, feature flags, and per-tenant overrides without reinventing the wheel.

Multi-tenant SaaS

Store per-organization, per-client and per-user settings with a clear cascade model. Roll out differentiated plans and entitlements safely.

Feature flags & experiments

Toggle features globally, for specific clients, or for a single user. Query everything via a single external API with permission-aware access.

Configuration hub for services

Use DynamicAuth, JWT helpers and permission filters to expose settings safely to CLIs, backends, and frontends over a single REST surface.

Quickstart

From zero to running in minutes

The service ships with demo scripts, a CLI and a full admin UI. For detailed steps see START-HERE.md.

  1. 1 Install dependencies: npm install
  2. 2 Copy env: cp .env.example .env and set MONGODB_URI, SESSION_SECRET.
  3. 3 Start MongoDB locally or via Docker, then run npm start (or npm run dev).
  4. 4 Open the admin UI at http://localhost:3000 (or your configured PORT) and log in with the admin credentials from .env.
  5. 5 Run ./test.sh or ./demo.sh to see the full workflow.
End-to-end demo
# Start the service
npm start

# In another terminal, run the demo
./demo.sh

# Demo will:
- create an organization
- add global/client/user/dynamic settings
- configure DynamicAuth (JS-based)
- exercise cascade + permission filters

Architecture

Two API layers, one source of truth

Internal UI traffic uses session-based auth under /api/internal/*, while external clients integrate via DynamicAuth-secured endpoints under /api/*.

  • • Multi-tenant data model (organizations, global/client/user/dynamic settings).
  • • Cascade resolution with LRU cache for fast reads.
  • • DynamicAuth (HTTP & JS) with JWT helpers and permission filters.
  • • CLI tool that talks to the same external API surface.

Security

Built-in safety rails

The reference implementation includes rate limiting, secure sessions, CORS, and sandboxed JS execution with vm2. Permission filters are enforced at query and record level.

External request shape

GET /api/settings/global HTTP/1.1
Authorization: Bearer <token>
X-Organization-Id: <org-id>
X-Auth-Name: default
